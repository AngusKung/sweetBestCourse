      哪一學年度修課： 103-1

      ψ 授課教師 (若為多人合授請寫開課教師，以方便收錄) 黃鐘揚

      δ 課程大概內容

                00. Class Introduction
                01. Data Structure in Programming
                02. Programming on Linux Workstations
                03. C++ Review (Variable)
                04. C++ Review (Class)
                05. C++ Review (Overloading and Polymorphism)
                06. C++ Review (I/O Libraries and Exception Handling)
                07. Memory Management
                08. Software Engineering
                09. Computational Complexity
                10. Dynamic Array and Linked List
                11. Tree
                12. Graph and Circuit
                13. Heap, Set, and Map
                14. Cache and Hash
                15. Final Project Discussion (I)
                16. Final Project Discussion (II)
                17. Special Topic

      Ω 私心推薦指數(以五分計) ★★★★★

                只是想跟團大家一起修：★
                熱愛寫程式：★★★★★

      η 上課用書(影印講義或是指定教科書)

                無指定用書

      μ 上課方式(投影片、團體討論、老師教學風格)

                使用投影片上課，就是跟著投影片講。通常老師會在一個topic講很
                久，加上示範寫code的時間，常常會超過下課時間，而拖到下個禮
                拜才會講完進度。因為作業是兩週一次，所以出新作業的那週都會
                留將近一堂半的課來講解作業。

      σ 評分方式(給分甜嗎？是紮實分？)

                今年的配分方式是：
                    作業 70%
                    project 30%
                HW1到HW3與HW7配分較輕，HW4到HW6占的分數大概會是前面的兩倍

                評分方式就是用測資評分，但不會公佈測資，所以自己要多多測試
                如果有神人同學一起修的話，可以拜託他們，借用他們生的測資
                另外，compile一定要過，最好再三確認甚至在其他人的電腦上跑跑看
                不然助教compile如果有error，成績會不太好看。

      ρ 考題型式、作業方式

                七次作業，老師第一堂課就會簡單說明每次作業的內容以及code大概要
                寫的行數。
                沒有期末考，但有期末project，due date是在期末考完隔週的星期二
                晚上，所以大家會在期末考完一起在系K一起coding，也就是所謂的
                DSnP camp。


      ω 其它(是否注重出席率？如果為外系選修，需先有什麼基礎較好嗎？老師個性？
加簽習慣？嚴禁遲到等…)

                好像是一類加選，只要上網登記就好。這學期在明達231上課，教室
                有190個位子，但選課人數有220左右。後來退選完之後也還有210個，
                所以期中前幾乎都有十幾個人需要坐走道。建議要提早到，坐前面一點
                的位子。外系選修建議要有C++基礎。

      Ψ 總結

                實在太多人選這堂課了，期中之前堂堂爆滿，空氣很差。不過期中
                之後作業開始變重了，停修潮出現，最後大概2成的人停修。建議
                是真的對寫code有興趣，而且學分數不要超過20學分，千萬千萬不要
                看到大家都修就跟團，不然會很痛苦，而且最後還是會需要停修。
                這學期還蠻多大二的來修，不過 真 的 很 累，必修課會hold不住，
                不要輕易嘗試。
                作業部分，我覺得前三次作業有點無聊，只是給大家複習c++語言用的。
                到作業四才會感覺到在寫有趣的東西。期末project其實比較像一次
                很大的作業，不過內容比作業有趣很多。寫一次作業，大概會花掉一個
                週末，後面幾個作業每個大概要花十幾個小時甚至到二十個小時。期末
                project要提前開始，先把前面簡單的寫完，架構先寫好，不然
                就算期末考完的四天幾乎不睡，也不見得寫得完。總之，要修的話，要
                抱著花9學分的力氣拿3學分的準備。會很有收獲的。
